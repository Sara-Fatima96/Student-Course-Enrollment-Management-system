# 🎓 Student Course Enrollment Management System (SQL)

## 📌 Project Overview
The **Student Course Enrollment Management System** is a SQL-based database project developed to efficiently manage student records, course details, instructor information, and enrollments for a training institute.

The project replaces manual Excel-based record management with a structured relational database, making it easier to store, retrieve, and analyze data using SQL queries.

## 🎯 Business Problem
As the number of students increases, maintaining student records in Excel becomes difficult and time-consuming. Manual tracking often leads to errors, duplicate records, and inefficient reporting.

## 🎯 Objective
- Design a relational database for a training institute.
- Store student, course, instructor, and enrollment information.
- Perform SQL queries to generate business insights.
- Improve data organization and reporting.

## 🛠️ Tech Stack

- MySQL
- SQL (DDL & DML)
- Relational Database Design

## 🗂️ Database Schema

The project consists of four tables:

- **Students**
- **Courses**
- **Instructors**
- **Enrollments**

### Relationships
- One Instructor → Multiple Courses
- One Student → Multiple Enrollments
- One Course → Multiple Student Enrollments

## 📊 Business Questions Solved

- Total number of students
- Total number of courses
- Total number of instructors
- Courses with fees greater than 5000
- Student count for each course
- Students enrolled in each course
- Students who completed their courses
- Students currently pursuing courses
- City-wise student distribution
- Revenue generated by each course

## 📈 Key Insights

- The institute has **100 students**.
- A total of **5 courses** are offered.
- There are **5 instructors**.
- **Data Analytics** has the highest student enrollment.
- **Hyderabad** has the highest number of enrolled students.
- **Data Analytics** generates the highest revenue.
- **Excel** generates the lowest revenue.
